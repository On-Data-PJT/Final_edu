# Components

Last Updated: 2026-04-08

## Purpose

이 문서는 웹 사이트에 들어갈 화면 구성요소를 페이지 단위로 정리하는 UI 명세 문서입니다.

- 현재는 아래 3개 화면을 정의합니다.
  - `Page 1`: 자료 등록 메인 페이지
  - `Page 2`: 첫 번째 결과 페이지
  - `Page 3`: 솔루션 페이지
- 이 문서는 구현용 체크리스트이므로, 단순 아이디어가 아니라 **표시 요소, 상호작용, 상태 변화, 데이터 의존성**까지 적습니다.
- 구조와 상호작용은 이 문서를 따르고, 시각 표현은 `.agent/DESIGN.md`를 따릅니다.

## Global Direction

- 메인 페이지의 배치 기준은 사용자가 제공한 벤치마크 이미지의 정보 밀도와 구성 리듬을 참고합니다.
- 특히 아래 배치를 유지합니다.
  - 좌측 상단의 아주 작은 텍스트 로고
  - 우측 상단의 작은 아이콘 액션
  - 화면 정중앙의 주요 입력 영역
- 결과 페이지는 한 화면 안에서 연속적으로 정보를 읽는 **스크롤형 분석 리포트**로 구성합니다.
- 이 문서는 구조와 상호작용을 정의하고, `.agent/DESIGN.md`는 그것이 어떻게 보여야 하는지를 정의합니다.

## Shared Data Contracts

- `과정(course)`은 아래 데이터를 가집니다.
  - 과정명
  - 원본 커리큘럼 PDF
  - 추출된 대주제 목록
  - 각 대주제의 목표 비중
- 목표 비중은 아래 순서로 확보합니다.
  - 커리큘럼 PDF에서 시간, 기간, 주차, 퍼센트, 시수 같은 수치 정보 자동 추출 시도
  - 추출 결과를 `Add Course` 모달 안에서 표 형태로 보여주기
  - 사용자가 직접 수정 후 저장
- 결과 페이지의 `view mode`는 아래 3개로 고정합니다.
  - `only발화`: YouTube 자막 또는 STT로 얻은 발화 텍스트만 사용
  - `only자료`: 업로드 파일 텍스트만 사용
  - `발화+자료`: 두 소스를 함께 사용
- `view mode` 토글은 `Page 2`의 3번, 4번 섹션에만 적용됩니다.

## Page 1. Main Page

### 1. Header

- 좌측 상단에는 매우 심플한 텍스트 로고 `Final_Edu`를 작게 배치합니다.
- 우측 상단에는 작은 아이콘 2개를 배치합니다.
  - 첫 번째 아이콘: `책 +` 의미의 과정 추가 아이콘
  - 두 번째 아이콘: `목록/라이브러리` 의미의 과정 목록 아이콘
- 헤더는 설명 문구 없이 최대한 간결하게 유지합니다.

### 2. Add Course Trigger

- `책 +` 아이콘을 클릭하면 작은 윈도우형 모달이 열려야 합니다.
- 이 모달은 새 과정을 생성하는 용도입니다.

### 3. Add Course Modal

- 모달 안에는 아래 항목이 반드시 있어야 합니다.
  - `과정명` 텍스트 입력 필드
  - `커리큘럼 PDF` 단일 업로드 드롭박스
- 과정명은 사용자가 직접 텍스트로 입력합니다.
- 커리큘럼 파일은 **PDF 1개만 허용**합니다.
- 드롭박스는 클릭 업로드와 drag-and-drop 둘 다 허용하는 방식이 바람직합니다.
- PDF가 업로드되면 시스템은 해당 PDF에서 커리큘럼을 추출해 아래를 파악해야 합니다.
  - 과정의 전체 교육 흐름
  - 대단원 또는 주요 교육 단계
  - 각 대주제의 목표 비중 후보
- 목표 비중 후보는 아래 정보가 있으면 우선 활용합니다.
  - 강의 시간
  - 강의 기간
  - 주차
  - 시수
  - 목표 퍼센트
- 자동 추출 후 모달 안에는 `대주제 / 설명 / 목표 비중` 표가 추가로 보여야 합니다.
- 사용자는 각 대주제의 목표 비중을 모달 안에서 직접 수정할 수 있어야 합니다.
- 자동 추출이 불완전해도 저장을 막지 않고, 사용자가 수정 가능한 초안 상태를 만들어야 합니다.
- 모달의 기본 상태는 아래처럼 정의합니다.
  - 초기 상태: 과정명 비어 있음, PDF 없음
  - 파일 선택 상태: PDF 1개가 연결됨
  - 추출 중 상태: 커리큘럼 분석 진행
  - 추출 성공 상태: 과정 생성 가능, 목표 비중 수정 가능
  - 추출 실패 상태: 오류 안내 후 재업로드 가능
- 모달 하단에는 최소 아래 액션이 필요합니다.
  - `취소`
  - `저장`

### 4. Course List Trigger

- 우측 상단의 목록 아이콘을 클릭하면 기존에 추가한 과정들을 볼 수 있는 작은 창이 열려야 합니다.
- 이 창은 화면 전체를 덮는 페이지가 아니라, **작은 팝오버/패널** 형태가 적합합니다.

### 5. Course List Panel

- 이 패널에는 기존에 추가된 과정 및 커리큘럼이 간단히 보여야 합니다.
- 각 항목에는 최소 아래 정보가 있어야 합니다.
  - 과정명
  - 커리큘럼 추출 완료 여부
  - 대주제 수 또는 간단한 요약
  - 선택 가능 상태
- 사용자가 특정 과정을 클릭하면, 해당 과정이 현재 활성 과정이 됩니다.
- 현재 활성 과정이 무엇인지 패널 안에서 구분 가능해야 합니다.

### 6. Main Center Workspace

- 메인 페이지 정중앙에는 주요 입력을 위한 적당한 크기의 드롭박스를 둡니다.
- 이 영역은 **선택된 과정이 있을 때 강사별 자료를 등록하는 메인 워크스페이스**입니다.
- 과정이 아직 선택되지 않았더라도 중앙 박스는 화면에 보이되, 비활성 상태로 둡니다.
- 비활성 상태에서는 아래 수준의 안내 메시지를 보여줍니다.
  - `과정을 선택하거나 새로 추가하세요`
  - `과정이 선택되면 강사 자료를 등록할 수 있습니다`

### 7. Instructor Submission Block

- 중앙 워크스페이스의 기본 단위는 `강사 입력 블록`입니다.
- 첫 번째 과정이 선택되면 기본적으로 강사 입력 블록 1개를 표시합니다.
- 각 강사 입력 블록은 아래 요소로 구성합니다.
  - 좌측: `강사명` 텍스트 입력 필드
  - 중앙: 해당 강사의 강의 자료 등록용 드롭박스
  - 드롭박스 하단: 작은 액션 아이콘 2개
- 강사명은 사용자가 직접 텍스트로 입력합니다.
- 강의 자료 드롭박스는 선택된 과정의 커리큘럼 기준으로 비교될 자료를 담는 역할입니다.

### 8. Instructor File Upload Action

- 강사 블록 하단 좌측의 작은 아이콘은 `파일 업로드` 버튼입니다.
- 클릭 시 강사 자료 파일을 등록할 수 있는 작은 기능 창이 열려야 합니다.
- 현재 MVP 기준으로 지원 대상은 아래 포맷을 기본값으로 둡니다.
  - `PDF`
  - `PPTX`
  - `TXT`
  - `MD`
- 파일이 등록되면 해당 강사 블록 안에서 자료가 추가되었다는 상태가 보여야 합니다.

### 9. Instructor YouTube Upload Action

- 강사 블록 하단 우측의 작은 아이콘은 `YouTube 링크 업로드` 버튼입니다.
- 클릭 시 YouTube 링크를 입력할 수 있는 작은 기능 창이 열려야 합니다.
- 이 창은 강사별 YouTube 강의 링크를 등록하는 용도입니다.
- 입력 방식은 한 줄에 하나씩 URL을 적는 구조를 기본값으로 둡니다.
- 링크가 등록되면 해당 강사 블록 안에서 링크가 추가되었다는 상태가 보여야 합니다.

### 10. Submission Controls

- 중앙 드롭박스 아래에는 작은 `+` 버튼과 작은 `enter-like` 아이콘 버튼을 나란히 둡니다.
- `+` 버튼은 새로운 강사 입력 블록을 추가하는 용도입니다.
- 클릭 시 현재 입력 중이던 블록들은 위로 살짝 올라가고, 아래에 새 강사 입력 블록이 생성되어야 합니다.
- 새로 생성되는 블록은 아래 상태로 시작합니다.
  - 강사명 비어 있음
  - 업로드된 파일 없음
  - 등록된 YouTube 링크 없음
- `enter-like` 아이콘 버튼은 **강사 추가와 자료 업로드가 끝났음을 확정하고 다음 페이지로 넘어가는 버튼**입니다.
- 이 버튼은 아래 조건이 충족될 때만 활성화합니다.
  - 과정이 선택됨
  - 최소 2명 이상의 강사 블록이 유효함
  - 각 유효 강사 블록에 강사명과 자료가 있음

### 11. Empty / Disabled State

- 과정이 선택되지 않은 상태에서는 아래 기능을 모두 비활성화합니다.
  - 강사 자료 드롭박스
  - 파일 업로드 버튼
  - YouTube 링크 버튼
  - 강사 추가 버튼
  - 다음 페이지 이동 버튼
- 단, 비활성 상태여도 화면 구조 자체는 유지합니다.

### 12. Interaction Flow

1. 사용자는 우측 상단의 `책 +` 아이콘을 눌러 새 과정을 추가합니다.
2. 과정명과 커리큘럼 PDF를 등록합니다.
3. 시스템은 PDF에서 커리큘럼과 목표 비중 후보를 추출합니다.
4. 사용자는 모달 안에서 목표 비중을 확인하고 필요 시 수정합니다.
5. 과정 생성이 완료되면 해당 과정은 과정 목록에 추가됩니다.
6. 사용자는 목록 아이콘에서 과정을 선택합니다.
7. 메인 중앙 워크스페이스가 활성화됩니다.
8. 사용자는 강사명을 적고 파일 또는 YouTube 링크를 등록합니다.
9. 필요 시 `+` 버튼으로 강사 입력 블록을 추가합니다.
10. `enter-like` 버튼으로 결과 페이지로 이동합니다.

## Page 1 Acceptance Checklist

- 좌측 상단에 작은 `Final_Edu` 로고가 있어야 합니다.
- 우측 상단에 `책 +` 아이콘과 `목록` 아이콘이 있어야 합니다.
- `책 +` 아이콘 클릭 시 과정명 입력 + PDF 1개 드롭존이 있는 작은 모달이 떠야 합니다.
- 과정 커리큘럼 파일은 PDF 하나만 허용해야 합니다.
- 커리큘럼 PDF 분석 후 목표 비중 표가 모달 안에 표시되고 수정 가능해야 합니다.
- 목록 아이콘 클릭 시 기존 과정들을 확인할 수 있는 작은 창이 떠야 합니다.
- 과정을 선택하면 중앙 입력 영역이 활성화되어야 합니다.
- 중앙 영역에는 강사명 입력 필드, 자료 드롭박스, 파일 업로드 아이콘, YouTube 업로드 아이콘이 있어야 합니다.
- 중앙 영역 하단에는 강사 블록을 추가하는 작은 `+` 버튼과 다음 페이지 이동용 `enter-like` 버튼이 있어야 합니다.
- 과정이 선택되지 않으면 중앙 입력 영역은 보이되 비활성 상태여야 합니다.

## Page 2. Result Page 1

### 1. Page Role

- 이 페이지는 첫 번째 결과 페이지이며, 분석 결과를 **조작보다 읽기 중심의 스크롤형 리포트**로 보여줍니다.
- 페이지는 크게 위에서 아래로 `1번~4번`의 네 개 섹션으로 구성합니다.

### 2. Top Utility Controls

- 우측 상단에는 작은 `view mode` 버튼 그룹을 둡니다.
- 이 버튼 그룹은 텍스트 대신 아이콘 중심으로 구성합니다.
  - `only발화`: 말하는 사람 느낌의 아이콘
  - `only자료`: 책 + PPT가 있는 화면 느낌의 아이콘
  - `발화+자료`: 자료 앞에서 설명하는 사람 느낌의 아이콘
- 이 토글은 아래 섹션에만 영향을 줍니다.
  - 3번 섹션의 평균/강사별 가로 바 그래프
  - 4번 섹션의 목표 대비 라인 그래프
- 1번 섹션과 2번 워드클라우드는 기본적으로 `발화+자료` 기준을 사용합니다.

### 3. Section 1. Curriculum Target vs Instructor Actual

- 이 섹션은 좌우 2단으로 구성합니다.
- 좌측에는 과정 커리큘럼 기준표를 둡니다.
  - 대주제명
  - 구성 비율
  - 목표 퍼센테이지
- 우측에는 선택 강사의 실제 강의 비중을 보여주는 나이팅게일 로즈 차트를 둡니다.
- 강사 선택 UI는 원형 프로필 또는 원형 이름 배지로 구성합니다.
- 원형 프로필 내부 또는 인접 위치에 좌우 이동 버튼을 두어 강사를 전환합니다.
- 강사를 전환하면 로즈 차트가 즉시 바뀌어야 합니다.

### 4. Section 2. Instructor Word Cloud

- 이 섹션은 1번 섹션에서 선택한 강사와 연동됩니다.
- 강사 전환 시 워드클라우드도 함께 바뀝니다.
- 워드클라우드는 강사가 많이 강조하는 개념, 자주 설명하는 키워드, 자주 언급한 주제를 보여줍니다.
- 목적은 강사마다 강조 지점이 어떻게 다른지 한눈에 비교하게 하는 것입니다.

### 5. Section 3. Coverage Composition Bars

- 이 섹션은 상하 2단으로 구성합니다.
- 상단에는 **전체 강사 평균 구성 비중**을 stacked horizontal bar 형태로 배치합니다.
- 하단에는 **강사별 구성 비중**을 stacked horizontal bar 목록으로 배치합니다.
- 각 대주제는 페이지 전체에서 동일한 색을 사용합니다.
- `view mode` 토글에 따라 이 섹션의 값이 바뀝니다.
  - `only발화`
  - `only자료`
  - `발화+자료`

### 6. Section 4. Goal vs Actual Line Comparison

- 마지막 하단 섹션에는 목표와의 비교를 라인 그래프로 표시합니다.
- 목표 커리큘럼 비중은 고정 라인으로 표시합니다.
- 우측에는 강사 선택 UI를 두고, 선택된 강사의 실제 비중 라인을 함께 그립니다.
- 강사 라인 색상은 강사마다 일관되게 다르게 유지합니다.
- 이 섹션도 `view mode` 토글에 따라 값이 바뀝니다.
- 우측 최하단에는 다음 페이지로 이동하는 작은 버튼을 둡니다.

## Page 2 Acceptance Checklist

- 결과 페이지는 단일 스크롤 페이지여야 합니다.
- 우측 상단에 `view mode` 로고 버튼 3개가 있어야 합니다.
- `view mode` 토글은 3번, 4번 섹션에만 영향을 줘야 합니다.
- 1번 섹션은 좌측 기준표, 우측 나이팅게일 로즈 차트로 나뉘어야 합니다.
- 강사 선택 시 로즈 차트와 2번 워드클라우드가 함께 바뀌어야 합니다.
- 3번 섹션은 평균 stacked bar와 강사별 stacked bar를 모두 보여야 합니다.
- 4번 섹션은 목표 라인과 선택 강사 라인을 비교해 보여야 합니다.
- 페이지 우측 최하단에는 솔루션 페이지로 이동하는 버튼이 있어야 합니다.

## Page 3. Solution Page

### 1. Page Role

- 이 페이지는 첫 번째 결과 페이지의 지표를 바탕으로 도출 가능한 **순수 인사이트**만 보여주는 페이지입니다.
- 솔루션 페이지는 차트를 반복하지 않고, 사람이 바로 읽을 수 있는 개선 포인트 카드 중심으로 구성합니다.

### 2. Layout

- 세로 스크롤형 카드 목록으로 구성합니다.
- 인사이트는 기본적으로 `5~6개`를 목표로 합니다.
- 각 카드에는 전구 또는 의미에 맞는 아이콘을 작게 사용합니다.
- 카드마다 아래 구조를 권장합니다.
  - 짧은 제목
  - 무엇이 문제인지 한 줄 요약
  - 왜 그런 판단이 나왔는지 근거 요약
  - 실무적 개선 제안

### 3. Insight Cards

- 인사이트 1
  - 목표 커리큘럼과 비교해 강사별 과목당 강의 비중이 맞지 않는 경우의 개선 포인트
- 인사이트 2
  - 전체 강사 평균과 비교해 특정 강사의 비중 편차가 유난히 큰 경우의 개선 포인트
- 인사이트 3
  - 워드클라우드 기준으로 정의된 커리큘럼에 없는 내용을 개별적으로 많이 다루는 강사가 있는 경우의 개선 포인트
- 인사이트 4
  - 전체 커리큘럼 목표치와 전체 강사 평균이 크게 어긋나는 경우, 커리큘럼 자체가 현장과 동떨어졌을 가능성에 대한 개선 포인트
- 인사이트 5
  - 여러 강사가 공통적으로 다루지만 커리큘럼에는 없는 주제가 있다면 이를 반영하자는 개선 포인트
- 인사이트 6
  - 구현 가능 시, 최근 동향/화제 기법/타 기관 공통 과목을 참고한 커리큘럼 개선 포인트

### 4. External Trend Insight Handling

- 인사이트 6은 다른 카드들과 달리 외부 조사 데이터에 의존합니다.
- 이 카드는 아래 상태를 가져야 합니다.
  - 외부 조사 성공: 최신 동향 기반 제안 표시
  - 외부 조사 실패: 카드 전체를 숨기지 말고 `외부 동향을 충분히 수집하지 못했습니다` 같은 안내 문구 표시
- 이 인사이트는 핵심 분석 경로를 막지 않는 **보조 확장 인사이트**로 취급합니다.

## Page 3 Acceptance Checklist

- 솔루션 페이지는 차트보다 카드형 인사이트 중심으로 보여야 합니다.
- 기본적으로 5개 이상의 인사이트가 보여야 합니다.
- 각 인사이트는 문제, 근거, 개선 제안을 최소 단위로 가져야 합니다.
- 인사이트 1~5는 내부 분석 결과만으로 생성 가능해야 합니다.
- 인사이트 6은 외부 조사 실패 시에도 페이지 전체를 깨뜨리지 않아야 합니다.

## Implementation Notes

### 1. 바로 구현 가능한 범위

- Page 1의 `enter-like` 완료 버튼
- Page 2의 4단 스크롤 리포트 구조
- 강사 선택형 워드클라우드
- 평균/강사별 stacked bar
- 목표 대비 line comparison
- Page 3의 인사이트 1~5

### 2. 백엔드 확장이 필요한 범위

- `only발화`
  - 현재는 YouTube 자막 기반 데이터만 안정적으로 존재합니다.
  - 자막 없는 영상까지 다루려면 STT 파이프라인이 필요합니다.
- 목표 비중 자동 추출
  - PDF 형식 차이가 커서 자동 추출만으로는 부족할 수 있습니다.
  - 최종 목표 비중은 사용자가 `Add Course` 모달에서 보정하는 구조가 전제입니다.

### 3. 고위험 범위

- YouTube 자막이 없는 영상에 대한 STT
  - 현재 저장소에는 `yt-dlp`, `ffmpeg`, STT 배치 경로가 아직 없습니다.
  - 구현 시 worker 기준의 비동기 처리와 실패 경고 설계가 필요합니다.
- 외부 조사 기반 인사이트 6
  - 일반 웹 검색 전체를 대상으로 하면 신뢰도, 속도, 출처 관리 부담이 큽니다.
  - 따라서 인사이트 6은 핵심 경로가 아니라 확장 기능으로 다루는 것이 안전합니다.
